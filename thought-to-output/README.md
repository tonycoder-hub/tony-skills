# thought-to-output

把语音转文字、碎片笔记、口语化表达和重复发散的原始素材，整理成结构化、可阅读、可复用的内容资产。

## 目录说明

- `SKILL.md`：Skill 主定义文件
- `references/output-categories.md`：可输出类别说明
- `references/quality-checklist.md`：输出质量检查清单
- `assets/sample-outputs.md`：示例输入与示例输出

## 适用场景

- 整理语音转文字
- 把碎片想法沉淀成备忘录
- 把发散表达收敛成结构化内容
- 为后续写作、复盘或产品思考准备初稿

## 设计原则

- 先抽主线，再组织细节
- 只输出有依据支持的类别
- 无法稳定判断时，宁可不写
- 保留原始意图，不做过度拔高
