# Output Categories

这个文件定义 `thought-to-output` 在整理原始素材时可生成的输出类别。

## 核心原则

- 输出类别不是固定套餐
- 只生成被原始材料稳定支持的类别
- 如果某个类别依据不足，就不要生成
- **根据输入量级和性质，自动选择输出形态**

## 两种输出形态

### A. 结构化卡片

适用于短输入或有明确问题的中等输入。由以下类别组合而成。

### B. 完整成稿

适用于长输入（> 1000 字）、个人反思 / 叙事 / 日记类素材。输出为一篇完整文章，不拆分为卡片类别。

---

## 结构化卡片的可选类别

### 1. 主线一句话

适用于几乎所有输入。

作用：

- 用一句话概括用户真正想表达的核心
- 帮助后续所有输出围绕同一主线展开

### 2. 核心摘要

适用于存在多个要点、重复表达或主题已经出现但比较散乱的输入。

作用：

- 压缩重复内容
- 提取主要信息点
- 让用户快速看到重点

### 3. 结构化整理稿

适用于原始内容较长、较乱或存在明显层次但尚未成型的输入。

作用：

- 按逻辑重组内容
- 形成可读性更强的整理版本

### 4. 个人表达版

适用于偏主观感受、反思、自我表达、情绪体验类材料，但输入量级不足以支撑完整成稿时使用。

作用：

- 保留个人语气
- 让表达更连贯、更适合直接继续写作

注意：当输入足够长且偏个人叙事时，应直接走「完整成稿」路径，而非生成此类别。

### 5. 产品思考版

适用于输入明显围绕产品定位、用户价值、功能边界、复杂度权衡等主题。

作用：

- 提炼产品视角下的判断
- 把零散观察转成更清楚的思考框架

### 6. 标题候选

适用于内容已经有清楚主题，且用户后续可能要发文、存档或继续创作。

作用：

- 帮助内容命名
- 提供不同表达角度的切入点

成稿的标题格式：`YYYY.MM.DD 主题句`

### 7. 行动项

适用于输入中已经出现明确问题、待推进事项或下一步方向。

作用：

- 从表达材料中提取可执行内容
- 帮助用户推进思考到行动

### 8. 待追问问题列表

适用于输入中存在信息缺口、判断边界不清或需要继续澄清的情况。

作用：

- 把不确定性显式化
- 帮助用户继续补全上下文

---

## 完整成稿的要求

当判断需要输出完整成稿时，不使用上述卡片类别，而是直接生成一篇文章：

- **标题**：`YYYY.MM.DD 主题句`
- **正文**：第一人称、连贯叙事、保留细节、暗线串联多主题
- **结尾**：回扣开头，形成闭环
- **不附加**：摘要、行动项、追问清单等结构化卡片（除非用户明确要求）

详细要求见 `SKILL.md` 中的「完整成稿」部分。

---

## 选择建议

| 输入特征 | 推荐输出 |
|---------|---------|
| 短输入（< 300 字），主题单一 | 主线一句话 + 核心摘要 |
| 中等输入，有明确问题 | 结构化整理 + 行动项 |
| 长输入，个人反思 / 叙事 / 日记 | **完整成稿** |
| 长输入，偏产品 / 工作 / 复盘 | 产品思考版 + 行动项 |
| 素材不足 | 主线一句话 + 待追问问题 |
