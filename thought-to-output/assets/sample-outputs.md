# Sample Outputs

这个文件提供 `thought-to-output` 的示例输入和输出风格参考。

---

## 示例一：短输入 → 结构化卡片（产品反思）

### 输入

"我最近一直觉得这个播放器越做越重，一开始只想把本地播放做好，后来又接了 SMB、WebDAV、DLNA。功能确实多了，但我越来越说不清楚它到底最核心是个什么产品。现在每次改动都像是在一个越来越复杂的系统上补丁。"

### 输出

**主线一句话**

这个问题的核心不是功能变多，而是产品边界扩张后，核心价值表达开始变得模糊。

**核心摘要**

- 产品从本地播放工具逐步扩展成更完整的媒体访问工具
- 功能增加带来能力提升，也带来更高的复杂度
- 当前焦虑点是产品定位模糊和维护成本上升

**行动项**

- 用一句话重写当前产品定义
- 拆分核心能力和增强能力
- 为后续新增功能设定收敛原则

## 示例二：短输入 → 结构化卡片（个人反思）

### 输入

"我感觉最近很多事不是不会做，而是我脑子里同时挂着太多线程。每件事都在推进，但没有一件事是很完整地推进，所以每天都觉得很忙，但晚上回头看又觉得没真正完成什么。"

### 输出

**主线一句话**

问题不在执行能力，而在注意力被过多并行事务切碎，导致推进感被削弱。

**个人表达版**

我最近的卡点不是不会做事，而是脑子里同时挂着太多线程。每一件事都在往前挪一点，但没有哪一件事被完整地推进下去。结果就是，白天一直在忙，晚上回头看，却很难说清楚自己真正完成了什么。这种疲惫感，不完全来自工作量，而更像来自持续切换和持续分散。

**待追问问题**

- 哪些线程其实可以暂停
- 哪些任务必须进入连续推进状态
- 每天"完成"的定义是否过于模糊

## 示例三：素材不足时的保守输出

### 输入

"这个方向可能不太对，但我还没想清楚。"

### 输出

**主线一句话**

用户对当前方向产生了怀疑，但还没有形成足够清楚的判断依据。

**待追问问题**

- 具体是哪一部分让你觉得方向不对
- 这种"不对"是来自结果、过程，还是资源投入
- 目前已经出现了哪些信号

说明：不生成行动项、洞察型总结或长篇成稿。原因是原始材料不足以稳定支持这些类别。

## 示例四：长输入 → 完整成稿（个人叙事）

### 输入（节选，实际输入约 5000 字口语转文字）

"我最近越来越强烈地意识到一件事，就是当脑子里想法很多很乱很丰沛的时候，只靠想是不够的。很多念头在脑子里盘旋的时候会给人一种错觉，好像自己已经想清楚了，但真正开始说出来写下来之后才发现，原来很多东西只是有感觉，还远远没到成形的程度……（中间涉及写作习惯、人际分散、钢铁侠 3 的叙事方式、25 岁的自我认知、消费行为分析、品牌信任、购物体验、满足感等多个主题）……也许这次整理并不能立刻得出一个完美结论，但至少让我更确定了一点，写下来不只是为了表达。"

### 输出

**2026.04.07 把想法写下来，也是在重新认识自己**

我最近越来越强烈地意识到一件事：当脑子里的思绪很多、很乱、很丰沛的时候，只靠"想"，其实是不够的。

很多念头在脑子里盘旋的时候，会给人一种错觉，好像自己已经想得很清楚了。但真正开始把它们说出来、写下来之后，才会发现，原来很多东西只是"有感觉"，还远远没有到"成形"的程度。反过来说，一旦你开始记录，无论是打字，还是语音转文字，那个过程本身就会逼着你去整理、归纳、收敛，最后把原本漂浮的想法变成一个相对完整的观点。

（……正文延续，保留所有主题和具体细节：2021-2022 年的写作习惯、人际关系分散注意力、《钢铁侠 3》的讲述感、25 岁的自我认知、年终奖后的消费行为、CD/Walkman/Contax T2/徕卡的消费链路、品牌信任的迁移逻辑、购物体验中的"目标感"、满足感的调节价值……）

也许这次整理并不能立刻得出一个完美结论，但至少它让我更确定了一点：写下来，不只是为了表达，更是为了理解；不只是为了记录发生了什么，也是为了慢慢看清楚，自己到底在乎什么，想成为什么样的人，又究竟希望这个世界以什么方式来回应自己。

### 成稿特征说明

- 输出形态：完整文章（约 3000-4000 字）
- 标题格式：日期 + 主题句
- 全篇第一人称，保留用户语气
- 多主题通过暗线"通过记录和消费来认识自己"串联
- 具体细节（品牌、影视、金额、产品型号）全部保留
- 结尾回扣开头（"写下来"的价值）
- 不附加结构化卡片
